Case Study

IMMO ETL

Pipeline Zillow ZHVI -> Postgres, concu pour etre idempotent et exploitable en pilotage metier.

Python PostgreSQL SQLAlchemy Metabase
Retour au portfolio

Probleme et objectif

Contexte

Les donnees immobiliere sont disponibles en CSV volumineux et doivent etre structurees pour analyses temporelles et comparaisons geographiques.

Objectif

Livrer un pipeline stable qui supporte les relances sans doublons, expose des metriques MoM/YoY et alimente un dashboard pour suivi marche.

Architecture

Extract

Lecture CSV + validation schema.

Transform

Wide to long, MoM/YoY, nettoyage.

Load

Upsert regions + upsert prices_monthly.

Visualize

Metabase dashboards de comparaison.

Decisions techniques

Idempotence

  • Cle primaire composite sur (region_id, dt).
  • Chargement via table temporaire puis ON CONFLICT DO UPDATE.

Fiabilite

  • Validation input CSV (RegionID + colonnes mensuelles).
  • Tests unitaires + integration DB optionnelle.
Voir une requete SQL representative 
SELECT dt, AVG(zhvi) AS avg_zhvi
FROM prices_monthly
GROUP BY dt
ORDER BY dt;

Impact

Resultat data

Pipeline relancable sans corruption, base analytique claire et requetes metier performantes.

Resultat produit

Dashboard IMMO Market Monitor avec tendances nationales et comparatifs etats.
